AMQP
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
AMQP (Advanced Message Queuing Protocol) — открытый протокол прикладного уровня для передачи сообщений между компонентами системы. Основная идея состоит в том, что 
отдельные подсистемы (или независимые приложения) могут обмениваться произвольным образом сообщениями через AMQP-брокер, который осуществляет маршрутизацию, возможно 
гарантирует доставку, распределение потоков данных, подписку на нужные типы сообщений.
1. Технология обмена сообщениями.
	1.1 Сообщение (messege) - единица передаваемых данных. Состоит из двух блоков: ключ маршрутизации, содержание (полезная нагрузка). Осноная его часть (содержание) 
	никак не интерпретируется сервером. К сообщению могут быть прицеплены структурированные заголовки.
		
	1.2 Точка обмена (exchange point) - в неё отправляются сообщения. Затем точка обмена распределяет сообщения в одну или несколько очередей (осуществляет 
	маршрутизацию по очередям). Сообщения распределенные по очередям затем отправляются потребителю.
		Типы точки обмена:
			* fanout(разветвление) - сообщение передается во все прицепленные к точке очереди
			* direct(непосредственный) - сообщения передаются в очередь с именем, совпадающим с ключом маршрутизации(routing key)
			* topic (тематический) - среднее между fanout и direct, сообщение будет отправленно в то колличество очередей которые будут совпадать по маске (регулярные
			выражения).
	1.3 Продюссер - программа, которая генерирует сообщения.
	1.4 Потребитель (consumer) - программа, которая принемает сообщения.
	1.5 Очередь(queue) - накопитель в котором хранятся сообщения до тех пор пока они не будут забраны клиентом.
	1.6 Брокер - программа управляющая циклом отправки сообщений от продюссера до потребителя. Например RabbitMQ, Kafka. Rabit доминирует в меньших масштабах, Kafka
		более для масштабирования.
